## 2. 架构与分层职责

### 2.1 分层架构

```text
api (接口层)
├── Resource: REST API 控制器
├── assembler: DTO/VO 转换器（MapStruct）
└── config: API 配置

domain (领域层)
├── model: 领域实体
├── service: 领域服务
└── repository: 仓储接口

infrastructure (基础设施层)
├── persistence: 持久化实现
│   ├── dao: MyBatis Mapper
│   └── *MyBatisRepository: Repository 实现
└── message: 消息定义

dto / vo / enums / security: 输入输出、枚举、安全认证
```

### 2.2 各层职责

- Resource 层
  - 接收 HTTP 请求、参数校验、调用 Service、返回 VO。
  - 必须有权限控制：`@PreAuthorize(...)`。
- Service 层
  - 业务编排、事务边界、调用 Repository。
  - 涉及写操作的方法必须加 `@Transactional(rollbackFor = Exception.class)`。
- Repository 层
  - 定义仓储接口与持久化实现，屏蔽 DAO 细节。
- DAO 层
  - 只负责 SQL/Mapper，不承载业务逻辑。

### 2.3 依赖方向（禁止反向依赖）

- 允许：`api -> domain -> infrastructure`
- 禁止：
  - Service 直接依赖 Controller
  - Service 直接编排 SQL（应经 Repository/DAO）
  - Controller 写业务事务

## 3. 数据库与实体规范

### 3.1 命名与主键

- 表名：系统表优先 `sys_` 前缀，统一 `snake_case`。
- 主键：统一 `id`，Java 类型统一 `BigInteger`。

### 3.2 审计与逻辑删除

- 所有实体必须继承审计基类：
  - `AbstractAutoGeneratedIdAuditable<BigInteger>` 或
  - `AbstractAutoGeneratedIdLogicalDeletionAuditable<BigInteger, Boolean>`
- 逻辑删除优先于物理删除。
- 逻辑删除字段：`deleted` / `is_deleted`（`Boolean`/`TINYINT(1)`）。

### 3.3 字段映射约定

- JSON 字段必须显式 `typeHandler`。
- 非表字段必须标注 `@TableField(exist = false)`。
- 布尔值字段推荐显式映射，避免语义歧义。

### 3.4 索引与约束

- 高频查询字段必须建索引。
- 业务唯一字段必须加唯一约束（可结合逻辑删除字段做联合唯一）。

## 4. i18n 与异常规范

### 4.1 i18n 基本要求

- 所有业务异常统一使用 `AuthCenterMessages` 枚举，不允许硬编码中英文错误文案。
- 新增错误码时，必须同步维护：
  - `silence_auth_center_messages.properties`
  - `silence_auth_center_messages_en.properties`
  - `silence_auth_center_messages_zh_CN.properties`

### 4.2 错误码规划

- 用户管理：51-60
- 角色管理：61-70
- 菜单管理：71-80
- 认证授权：81-90
- 其他业务：91+

### 4.3 异常处理规范

- 禁止吞异常（空 catch）。
- 禁止仅打印日志不抛出。
- 必须保留异常链：`throw new XxxException(msg, e)`。

## 5. 开发红线（必须遵守）

### 5.1 数据访问

- 禁止循环内逐条查库（N+1）。
- 禁止无分页拉全表数据。
- 禁止默认依赖 `select *`。

### 5.2 事务

- 写操作必须显式事务。
- 跨表一致性操作必须同一事务内完成。

### 5.3 安全

- 禁止明文存储密码，必须加密。
- 禁止日志输出敏感信息（密码、密钥、签名、nonce、token）。
- 接口返回 VO，禁止直接暴露实体。
- 所有 API 必须有权限注解。

### 5.4 代码质量

- 禁止魔法值、硬编码常量。
- DTO/Command 与领域对象转换必须使用 MapStruct。
- 查询对象优先使用 `@RelationalQueryProperty` + 转换器。

## 6. 常用模板

### 6.1 分页查询模板

```java
@GetMapping(value = "/users", params = {"pageNo", "pageSize"})
public Page<User> query(Page<User> page, UserQuery query) {
    var queryWrapper = QueryWrapperConverter.convert(query, User.class);
    return userService.query(page, queryWrapper);
}
```

### 6.2 批量关联模板

```java
@Transactional(rollbackFor = Exception.class)
private void assignUserRoles(BigInteger userId, List<BigInteger> roleIds) {
    List<UserRole> userRoles = roleIds.stream()
        .map(roleId -> {
            UserRole userRole = new UserRole();
            userRole.setUserId(userId);
            userRole.setRoleId(roleId);
            return userRole;
        })
        .collect(Collectors.toList());
    userRoleDao.insertBatchSomeColumn(userRoles);
}
```

### 6.3 树构建模板

```java
private List<MenuDto> buildMenuTree(List<Menu> menus) {
    var menuMapper = Mappers.getMapper(MenuMapper.class);
    var menuDtos = menus.stream()
        .map(menuMapper::convertToDto)
        .collect(Collectors.toList());

    Map<BigInteger, List<MenuDto>> parentMap = menuDtos.stream()
        .collect(Collectors.groupingBy(MenuDto::getParentId));

    menuDtos.forEach(menu ->
        menu.setChildren(parentMap.getOrDefault(menu.getId(), new ArrayList<>())));

    return parentMap.getOrDefault(BigInteger.ZERO, new ArrayList<>());
}
```

### 6.4 参数校验模板

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    User existingUser = userDao.selectById(user.getId());
    if (existingUser == null || existingUser.getDeleted()) {
        throw AuthCenterMessages.USER_NOT_EXIST.createException();
    }

    if (!existingUser.getUsername().equals(user.getUsername())
        && isUsernameExists(user.getUsername())) {
        throw AuthCenterMessages.USERNAME_ALREADY_EXIST.createException();
    }

    userDao.updateById(user);
}
```
