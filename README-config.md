# silence-config-center

## 项目简介

`silence-config-center` 是一个基于 Spring Boot 的配置中心服务，提供配置项管理、发布、查询、签名认证与长轮询订阅能力。

## 技术栈

- Spring Boot Web
- Spring Security
- MyBatis-Plus
- MapStruct
- Nacos Discovery / Client
- Caffeine
- MySQL

## 项目结构

- `api`：接口层（Controller/Resource），负责参数接收、协议转换、请求编排入口
- `domain/model`：领域模型
- `domain/service`：领域服务与业务编排（克隆、同步、签名校验、长轮询通知）
- `domain/repository`：仓储接口定义
- `infrastructure/persistence`：仓储实现
- `infrastructure/persistence/dao`：MyBatis DAO 与 SQL
- `dto` / `vo` / `api/assembler`：输入输出模型与 MapStruct 转换

## 快速开始

### 前置要求

- JDK 17+（按你本地环境为准）
- Maven 3.8+
- MySQL 8+

### 启动步骤

1. 配置数据库与应用参数（`src/main/resources/application-dev.yml`）。
2. 执行构建：`mvn clean package`。
3. 本地启动：`mvn spring-boot:run`。

## 核心能力

- 配置项 CRUD 与分页查询
- 配置发布与发布历史
- 命名空间克隆/同步
- 客户端长轮询订阅与变更通知
- 基于签名的请求认证

## 开发规范（摘要）

完整规范请查看：`CLAUDE.MD`

### 架构模式

- 严格遵循 `api -> domain -> infrastructure` 分层。
- Controller 不写 SQL，不承载跨聚合业务事务。
- 业务编排放在 Service，数据访问细节放在 Repository/DAO。

### 数据库规范

- 表名使用 `config_` 前缀 + `snake_case` 命名。
- 实体主键统一 `BigInteger`，继承 `AbstractAutoGeneratedIdAuditable<BigInteger>`。
- 布尔字段优先使用 `is_xxx` 列并通过 `@TableField` 显式映射。
- 当前无 JSON 列 `typeHandler` 约定；如新增 JSON 列，必须显式配置 `typeHandler`。

### 工具栈使用约定

- 查询条件统一使用 MyBatis-Plus `QueryWrapper` / `LambdaQueryWrapper`。
- DTO/Command 与领域对象转换统一用 MapStruct。
- 统一使用 SLF4J 记录日志。

### 开发红线

- 禁止吞异常（空 `catch` 或只写注释）。
- 禁止 `e.printStackTrace()`。
- 禁止在循环中逐条查库（N+1）。
- 禁止记录敏感信息明文（如签名、密钥、nonce）。
- 统一使用 `CommonErrors` 输出错误语义。

## 常用 API 示例

以下示例默认服务地址为 `http://localhost:8080`。

### 1. 分页查询配置项

```bash
curl -G 'http://localhost:8080/api/v1/configItems' \
  --data-urlencode 'pageNo=1' \
  --data-urlencode 'pageSize=10'
```

### 2. 创建配置项

```bash
curl -X POST 'http://localhost:8080/api/v1/configItems' \
  -H 'Content-Type: application/json' \
  -d '{
    "configEnvironmentId": 10001,
    "namespaceId": "application.yml",
    "formatType": "YML",
    "type": "PRIVATE",
    "content": "server:\n  port: 8081",
    "operationType": "ADD"
  }'
```

### 3. 更新配置内容

```bash
curl -X PUT 'http://localhost:8080/api/v1/configItems/123/content' \
  -H 'Content-Type: application/json' \
  -d '{
    "content": "server:\n  port: 8082",
    "operationType": "UPDATE"
  }'
```

### 4. 发布配置

```bash
curl -X PUT 'http://localhost:8080/api/v1/configItemReleaseHistories/release' \
  -H 'Content-Type: application/json' \
  -d '{
    "releaseName": "release-2026-03-10",
    "configItemId": 123,
    "oldContent": "server:\n  port: 8081",
    "content": "server:\n  port: 8082",
    "releaseType": "NORMAL",
    "distributionStrategy": "IMMEDIATE_RELEASE"
  }'
```

### 5. 客户端订阅配置（长轮询）

以下接口需要签名认证头：`X-Ca-Key`、`X-Ca-Timestamp`、`X-Ca-Nonce`、`X-Ca-Signature`。

```bash
curl -G 'http://localhost:8080/api/v1/configItems/subscribe' \
  -H 'X-Ca-Key: <accessKey>' \
  -H 'X-Ca-Timestamp: <timestamp_ms>' \
  -H 'X-Ca-Nonce: <nonce>' \
  -H 'X-Ca-Signature: <signature>' \
  --data-urlencode 'env=dev' \
  --data-urlencode 'componentCode=payment-service' \
  --data-urlencode 'namespace=application.yml'
```

### 6. 命名空间克隆

```bash
curl -X POST 'http://localhost:8080/api/v1/configNamespaces/clone' \
  -H 'Content-Type: application/json' \
  -d '{
    "sourceEnvironmentId": 10001,
    "targetEnvironmentIds": [10002, 10003],
    "cloneMode": "OVERWRITE_FILES"
  }'
```

### 7. 命名空间同步

```bash
curl -X POST 'http://localhost:8080/api/v1/configNamespaces/sync' \
  -H 'Content-Type: application/json' \
  -d '{
    "sourceConfigItemId": 123,
    "targetEnvironmentId": 10002,
    "targetNamespaceIds": ["application.yml", "db.yml"],
    "conflictStrategy": "OVERWRITE_FILES"
  }'
```
