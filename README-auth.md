# Silence Auth Center - 项目开发规范文档

## 1. 架构模式

### 1.1 分层架构

本项目采用经典的 DDD（领域驱动设计）分层架构：

```
api (接口层)
├── Resource: REST API 控制器
├── assembler: DTO/VO 转换器 (使用 MapStruct)
└── config: API 配置

domain (领域层)
├── model: 领域实体
├── service: 领域服务
└── repository: 仓储接口

infrastructure (基础设施层)
├── persistence: 持久化实现
│   ├── dao: MyBatis Mapper
│   └── *MyBatisRepository: Repository 实现
└── message: 消息定义

dto: 数据传输对象
├── *Command: 命令对象 (新增/修改)
├── *Query: 查询对象
└── *DTO: 数据传输对象

vo: 视图对象
security: 安全认证
enums: 枚举定义
```

### 1.2 各层职责

#### Resource 层（接口层）

- **职责**：接收 HTTP 请求，参数验证，调用 Service，返回响应
- **命名规范**：`*Resource.java`
- **核心注解**：`@RestController`, `@RequestMapping("/api/v1")`
- **权限控制**：使用 `@PreAuthorize("hasAuthority('system:xxx:xxx')")`

**示例**：

```java
@RestController
@RequestMapping("/api/v1")
public class UserResource {
    private final UserMapper userMapper;
    private final UserService userService;

    @GetMapping("/users/{id}")
    @PreAuthorize("hasAuthority('system:user:list')")
    public User getUserById(@PathVariable BigInteger id) {
        return userService.findById(id);
    }
}
```

#### Service 层（领域服务）

- **职责**：业务逻辑处理，事务控制，调用 Repository
- **命名规范**：`*Service.java`
- **核心注解**：`@Service`, `@Transactional(rollbackFor = Exception.class)`
- **事务规范**：所有涉及数据修改的方法必须添加事务注解

**示例**：

```java
@Service
public class UserService {
    private final UserDao userDao;

    @Transactional(rollbackFor = Exception.class)
    public BigInteger create(User user) {
        // 业务逻辑
        userDao.insert(user);
        return user.getId();
    }
}
```

#### Repository 层（仓储）

- **接口定义**：`domain/repository/*Repository.java`
- **实现类**：`infrastructure/persistence/*MyBatisRepository.java`
- **核心注解**：`@Repository`
- **职责**：封装数据访问逻辑，屏蔽持久化技术细节

**示例**：

```java
// 接口
public interface UserRepository {
    User findById(BigInteger id);
    int create(User user);
}

// 实现
@Repository
public class UserMyBatisRepository implements UserRepository {
    private final UserDao userDao;

    @Override
    public User findById(BigInteger id) {
        return userDao.selectById(id);
    }
}
```

#### DAO 层（数据访问）

- **命名规范**：`*Dao.java`
- **核心注解**：`@Mapper`
- **技术栈**：MyBatis-Plus BaseMapper
- **自定义 SQL**：使用 `@Select`, `@Update`, `@Delete` 注解

**示例**：

```java
@Mapper
public interface UserDao extends BaseMapper<User> {
    @Select("select * from sys_user where username = #{username} and status = #{status}")
    User findByUsernameAndStatus(String username, Boolean status);
}
```

## 2. 数据库规范

### 2.1 表命名规则

- **系统表前缀**：`sys_` (如 `sys_user`, `sys_role`, `sys_menu`)
- **业务表前缀**：根据模块定义（如通知表使用 `notice`）
- **命名风格**：蛇形命名法 (snake_case)

### 2.2 主键设计

- **主键类型**：`BigInteger`
- **生成策略**：自动生成（继承 `AbstractAutoGeneratedIdAuditable`）
- **字段名**：统一使用 `id`

### 2.3 必填字段（审计字段）

#### 基础审计字段

所有实体类继承 `AbstractAutoGeneratedIdAuditable<BigInteger>` 或 `AbstractAutoGeneratedIdLogicalDeletionAuditable<BigInteger, Boolean>`，自动包含：

- `id`: 主键
- `created_date` / `createdDate`: 创建时间
- `created_by` / `createdBy`: 创建人
- `last_modified_date` / `lastModifiedDate`: 最后修改时间
- `last_modified_by` / `lastModifiedBy`: 最后修改人

#### 逻辑删除字段

- `deleted` / `is_deleted`: 逻辑删除标识（Boolean 类型，0-未删除，1-已删除）

### 2.4 字段类型映射

| Java 类型 | 数据库类型 | 示例字段 |
|----------|----------|---------|
| `BigInteger` | `BIGINT` | id, user_id |
| `String` | `VARCHAR` | username, name |
| `Boolean` | `TINYINT(1)` | status, deleted |
| `Map<String, Object>` | `JSON` | meta (菜单元数据) |
| `Enum` | `VARCHAR` | type (使用枚举值) |

### 2.5 JSON 字段处理

使用 MyBatis-Plus 类型处理器：

```java
@TableField(typeHandler = StringToMapHandler.class)
private Map<String, Object> meta;
```

### 2.6 字段存在性控制

非数据库字段使用 `@TableField(exist = false)` 标识：

```java
@TableField(exist = false)
private Set<BigInteger> roleIds;
```

## 3. 常用工具栈

### 3.1 核心依赖

#### ORM 框架

- **MyBatis-Plus**: 数据库访问增强工具
  - `BaseMapper`: 基础 CRUD 操作
  - `LambdaQueryWrapper`: 类型安全的查询构造器
  - `Page`: 分页对象

**使用示例**：

```java
// 分页查询
Page<User> page = new Page<>(pageNo, pageSize);
userDao.selectPage(page, new LambdaQueryWrapper<User>()
    .eq(User::getStatus, true)
    .orderByDesc(User::getCreatedDate));
```

#### 对象转换

- **MapStruct**: 对象映射框架
  - 接口继承 `Converter<Source, Target>`
  - 配置类：`AuthCenterMapStructSpringConfig`

**使用示例**：

```java
@Mapper(uses = AuthCenterMapStructSpringConfig.class)
public interface UserMapper extends Converter<UserCommand, User> {
    User convert(UserCommand userCommand);
    UserVo toUserVo(User user);
}
```

#### 安全认证

- **Spring Security**: 权限控制
- **JWT (Auth0)**: Token 生成与验证
  - 密钥：`SecurityConstants.TOKEN_SECRET`
  - 前缀：`Bearer `

#### 查询构造器

- **自定义 QueryWrapper 转换器**：`QueryWrapperConverter`
  - 支持 `@RelationalQueryProperty` 注解
  - 自动转换 Query 对象为 MyBatis-Plus 查询条件

**使用示例**：

```java
public class UserQuery {
    @RelationalQueryProperty(type = Part.Type.STARTING_WITH)
    private String username;

    @RelationalQueryProperty(type = Part.Type.SIMPLE_PROPERTY)
    private Boolean status;
}

// Controller 中使用
var queryWrapper = QueryWrapperConverter.convert(query, User.class);
userService.query(page, queryWrapper);
```

### 3.2 工具类

#### 集合工具

- **CollectionUtils**: 集合转换工具
  - `transformToList`: 列表转换

#### 树形结构

- **TreeFormatUtils**: 树形数据处理
  - `listToTree`: 列表转树形结构

## 3.3 国际化（i18n）

### 3.3.1 国际化配置

#### 支持语言

- 英文 (en)
- 简体中文 (zh_CN)

#### 配置文件位置

```
src/main/resources/messages/
├── silence_auth_center_messages.properties          # 默认英文
├── silence_auth_center_messages_en.properties       # 英文（备份）
└── silence_auth_center_messages_zh_CN.properties    # 简体中文
```

### 3.3.2 错误消息定义

所有系统错误消息定义在 `AuthCenterMessages` 枚举中，并通过 properties 文件实现国际化。

**示例**：

```java
public enum AuthCenterMessages implements ErrorCodedEnumMessageSourceResolvable {
    USER_NOT_EXIST(HttpStatus.BAD_REQUEST, 51),
    PASSWORD_NOT_CORRECT(HttpStatus.BAD_REQUEST, 52),
    USERNAME_ALREADY_EXIST(HttpStatus.BAD_REQUEST, 59),
    // ...
}
```

#### 对应的国际化文件

**英文** (silence_auth_center_messages.properties):

```properties
AuthCenterMessages.UserNotExist=User does not exist
AuthCenterMessages.PasswordNotCorrect=Password is incorrect
AuthCenterMessages.UsernameAlreadyExist=Username already exists
```

**中文** (silence_auth_center_messages_zh_CN.properties):

```properties
AuthCenterMessages.UserNotExist=用户不存在
AuthCenterMessages.PasswordNotCorrect=密码错误
AuthCenterMessages.UsernameAlreadyExist=用户名已存在
```

### 3.3.3 使用规范

#### 必须遵守

1. **所有业务异常必须使用国际化消息**

```java
// 正确做法
if (user == null || user.getDeleted()) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}

// 错误做法
if (user == null || user.getDeleted()) {
    throw new BusinessException("用户不存在"); // 硬编码字符串
}
```

2. **新增异常常量必须同时更新 properties 文件**

```java
// 1. 在 AuthCenterMessages.java 中添加
USERNAME_ALREADY_EXIST(HttpStatus.BAD_REQUEST, 59),

// 2. 在 silence_auth_center_messages.properties 中添加
AuthCenterMessages.UsernameAlreadyExist=Username already exists

// 3. 在 silence_auth_center_messages_zh_CN.properties 中添加
AuthCenterMessages.UsernameAlreadyExist=用户名已存在
```

3. **特定参数的异常消息使用方法参数**

```java
// 正确做法 - 参数在运行时传递
throw AuthCenterMessages.USER_NOT_EXIST.createException(username);

// 支持的参数方法
createException()                    // 无参
createException(Object arg)          // 单参
createException(Object... args)      // 多参
```

4. **错误代码范围规划**

| 模块 | 代码范围 | 示例 |
|------|---------|------|
| 用户管理 | 51-60 | USER_NOT_EXIST (51) |
| 角色管理 | 61-70 | ROLE_NOT_EXIST (57) |
| 菜单管理 | 71-80 | MENU_NOT_EXIST (54) |
| 认证授权 | 81-90 | TOKEN_INVALID (81) |
| 其他业务 | 91+ | 根据需要定义 |

#### 禁止事项

1. **不要硬编码中文或英文字符串**

```java
// 错误
throw new BusinessException("用户不存在");
throw new BusinessException("User not exist");

// 正确
throw AuthCenterMessages.USER_NOT_EXIST.createException();
```

2. **不要混合使用国际化和硬编码**

```java
// 错误混合
if (error) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}
if (other) {
    throw new BusinessException("其他错误"); // 这是硬编码
}

// 正确 - 全部使用国际化
if (error) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}
if (other) {
    throw AuthCenterMessages.OTHER_ERROR.createException();
}
```

3. **不要在代码中拼接错误消息**

```java
// 错误
String msg = "User " + username + " not found";
throw new BusinessException(msg);

// 正确
throw AuthCenterMessages.USER_NOT_EXIST.createException(username);
```

### 3.3.4 i18n 消息文件维护清单

- [ ] 新增异常常量时同时更新三个 properties 文件
- [ ] 英文和中文翻译保持一致的逻辑含义
- [ ] 错误代码范围不重复
- [ ] 保持 properties 文件的编码为 UTF-8 (含中文)
- [ ] 定期审查未使用的消息常量

**使用示例**：

```java
var menuDtos = menus.stream()
    .map(menuMapper::convertToDto)
    .collect(Collectors.toList());

    menu.setChildren(parentMap.getOrDefault(menu.getId(), new ArrayList<>())));
```

## 4. 开发红线

### 4.1 数据库操作

#### 禁止事项

1. **禁止在循环中查询数据库**

```java
// 错误示例
for (BigInteger userId : userIds) {
    User user = userDao.selectById(userId); // 每次循环都查库
}

// 正确示例
List<User> users = userDao.selectBatchIds(userIds); // 批量查询
```

2. **禁止不加分页查询大量数据**

```java
// 错误示例
List<User> users = userDao.selectList(null); // 查询所有数据

// 正确示例
Page<User> page = new Page<>(pageNo, pageSize);
userDao.selectPage(page, queryWrapper);
```

3. **禁止使用 `select *`（MyBatis-Plus 默认查询所有字段，需注意）**

```java
// 如需指定字段，使用 LambdaQueryWrapper
new LambdaQueryWrapper<User>()
    .select(User::getId, User::getUsername);
```

#### 必须遵守

1. **所有修改操作必须添加事务注解**

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    // 更新逻辑
}
```

2. **关联数据修改必须在同一事务中**

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    userDao.updateById(user);
    userRoleDao.delete(new LambdaQueryWrapper<UserRole>()
        .eq(UserRole::getUserId, user.getId()));
    // 重新分配角色
    assignUserRoles(user.getId(), user.getRoleIds());
}
```

### 4.2 异常处理

#### 禁止事项

1. **禁止吞掉异常**

```java
// 错误示例
try {
    // 业务逻辑
} catch (Exception e) {
    // 什么都不做
}

// 正确示例
try {
    // 业务逻辑
} catch (Exception e) {
    log.error("业务处理失败", e);
    throw new RuntimeException("业务处理失败", e);
}
```

2. **禁止捕获异常后仅打印日志不抛出**

```java
// 错误示例
try {
    userDao.insert(user);
} catch (Exception e) {
    log.error("用户创建失败", e);
    // 没有抛出异常，调用方无法感知
}

// 正确示例
try {
    userDao.insert(user);
} catch (Exception e) {
    log.error("用户创建失败", e);
    throw AuthCenterMessages.USER_CREATE_FAILED.createException("用户创建失败", e);
}
```

#### 必须遵守

1. **使用统一的业务异常**

```java
if (existingRole == null || existingRole.getDeleted()) {
    throw AuthCenterMessages.ROLE_NOT_EXIST.createException("角色不存在");
}
```

2. **事务方法必须声明 `rollbackFor = Exception.class`**

```java
@Transactional(rollbackFor = Exception.class)
public void delete(BigInteger id) {
    // 业务逻辑
}
```

### 4.3 代码规范

#### 禁止事项

1. **禁止硬编码字符串**

```java
// 错误示例
if ("admin".equals(username)) { }

// 正确示例
if (SecurityConstants.ADMIN_USERNAME.equals(username)) { }
```

2. **禁止使用魔法数字**

```java
// 错误示例
user.setStatus(1);

// 正确示例
user.setStatus(true); // 或使用枚举
```

3. **禁止在 Service 层直接注入 DAO**

```java
// 错误示例（Service 层直接注入 DAO）
@Service
public class UserService {
    private final UserDao userDao; // 应该注入 Repository
}

// 正确示例
@Service
public class UserService {
    private final UserRepository userRepository;
}
```

#### 必须遵守

1. **实体类必须继承审计基类**

```java
public class User extends AbstractAutoGeneratedIdLogicalDeletionAuditable<BigInteger, Boolean> {
    // 字段定义
}
```

2. **DTO 转换必须使用 MapStruct**

```java
@Mapper(uses = AuthCenterMapStructSpringConfig.class)
public interface UserMapper extends Converter<UserCommand, User> {
    User convert(UserCommand userCommand);
}
```

3. **查询对象必须使用 `@RelationalQueryProperty` 注解**

```java
public class UserQuery {
    @RelationalQueryProperty(type = Part.Type.STARTING_WITH)
    private String username;
}
```

4. **逻辑删除必须使用标志位，禁止物理删除**

```java
@Transactional(rollbackFor = Exception.class)
public void delete(BigInteger id) {
    User user = userDao.selectById(id);
    user.setDeleted(true);
    userDao.updateById(user);
}
```

### 4.4 安全规范

#### 禁止事项

1. **禁止明文存储密码**

```java
// 必须加密存储，本项目示例中使用默认密码需注意生产环境安全
user.setPassword(passwordEncoder.encode("123456"));
```

2. **禁止在日志中打印敏感信息**

```java
// 错误示例
log.info("用户登录：{}", user); // user 对象包含密码

// 正确示例
log.info("用户登录：username={}", user.getUsername());
```

3. **禁止直接返回实体类**

```java
// 错误示例
@GetMapping("/users/{id}")
public User getUser(@PathVariable BigInteger id) {
    return userService.findById(id);
}

// 正确示例
@GetMapping("/users/{id}")
public UserVo getUser(@PathVariable BigInteger id) {
    User user = userService.findById(id);
    return userMapper.toUserVo(user);
}
```

#### 必须遵守

1. **所有接口必须添加权限控制**

```java
@PreAuthorize("hasAuthority('system:user:list')")
@GetMapping("/users/{id}")
public User getUserById(@PathVariable BigInteger id) {
    return userService.findById(id);
}
```

2. **Token 验证必须使用统一的认证机制**

```java
// 使用 SilenceHallServerTokenAuthority 进行 Token 签发和验证
var token = silenceHallServerTokenAuthority.issueToken(principal);
```

## 5. 最佳实践

### 5.1 分页查询模板

```java
@GetMapping(value = "/users", params = {"pageNo", "pageSize"})
public Page<User> query(Page<User> page, UserQuery query) {
    var queryWrapper = QueryWrapperConverter.convert(query, User.class);
    return userService.query(page, queryWrapper);
}
```

### 5.2 批量操作模板

```java
@Transactional(rollbackFor = Exception.class)
private void assignUserRoles(BigInteger userId, List<BigInteger> roleIds) {
    List<UserRole> userRoles = roleIds.stream()
        .map(roleId -> {
            UserRole userRole = new UserRole();
            userRole.setUserId(userId);
            userRole.setRoleId(roleId);
            return userRole;
        }).collect(Collectors.toList());
    userRoleDao.insertBatchSomeColumn(userRoles);
}
```

### 5.3 树形结构构建模板

```java
private List<MenuDto> buildMenuTree(List<Menu> menus) {
    var menuMapper = Mappers.getMapper(MenuMapper.class);
    var menuDtos = menus.stream()
        .map(menuMapper::convertToDto)
        .collect(Collectors.toList());

    Map<BigInteger, List<MenuDto>> parentMap = menuDtos.stream()
        .collect(Collectors.groupingBy(MenuDto::getParentId));

    menuDtos.forEach(menu ->
        menu.setChildren(parentMap.getOrDefault(menu.getId(), new ArrayList<>())));

    return parentMap.getOrDefault(BigInteger.ZERO, new ArrayList<>());
}
```

### 5.4 参数校验模板

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    // 1. 检查实体是否存在
    User existingUser = userDao.selectById(user.getId());
    if (existingUser == null || existingUser.getDeleted()) {
        throw AuthCenterMessages.USER_NOT_EXIST.createException("用户不存在");
    }

    // 2. 业务规则校验
    if (!existingUser.getUsername().equals(user.getUsername())
        && isUsernameExists(user.getUsername())) {
        throw AuthCenterMessages.USERNAME_ALREADY_EXIST.createException("用户名已存在");
    }

    // 3. 执行更新
    userDao.updateById(user);
}
```

## 7. 注意事项

1. **所有数据库实体类必须继承审计基类**，自动填充创建时间、修改时间等字段
2. **逻辑删除优先于物理删除**，保证数据可追溯
3. **事务边界清晰**，Service 层方法为事务边界
4. **接口返回值使用 VO 对象**，避免直接暴露实体类
5. **查询条件使用 Query 对象**，配合 `@RelationalQueryProperty` 注解自动构建查询
6. **权限控制细粒度到按钮级别**，使用 `system:模块:操作` 格式
7. **密码等敏感信息必须加密存储**
8. **所有外部输入必须进行校验**
