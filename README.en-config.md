# silence-config-center

## Overview

`silence-config-center` is a Spring Boot based configuration center service that provides configuration management, release workflows, query APIs, signature authentication, and long-polling subscription.

## Tech Stack

- Spring Boot Web
- Spring Security
- MyBatis-Plus
- MapStruct
- Nacos Discovery / Client
- Caffeine
- MySQL

## Project Structure

- `api`: API layer (Controller/Resource), request handling and protocol conversion
- `domain/model`: domain models
- `domain/service`: domain services and business orchestration (clone, sync, signature checks, long-polling notifications)
- `domain/repository`: repository interfaces
- `infrastructure/persistence`: repository implementations
- `infrastructure/persistence/dao`: MyBatis DAOs and SQL
- `dto` / `vo` / `api/assembler`: input/output models and MapStruct converters

## Quick Start

### Prerequisites

- JDK 17+ (based on your local runtime requirement)
- Maven 3.8+
- MySQL 8+

### Run Locally

1. Configure DB and app settings in `src/main/resources/application-dev.yml`.
2. Build the project: `mvn clean package`.
3. Start locally: `mvn spring-boot:run`.

## Core Capabilities

- Config item CRUD and paged query
- Config release and release history
- Namespace clone and sync
- Client long-polling subscription and change notification
- Signature-based request authentication

## Development Conventions (Summary)

For full conventions, see: `CLAUDE.MD`

### Architecture

- Follow strict layered dependency: `api -> domain -> infrastructure`.
- Controllers must not contain SQL or cross-aggregate transaction orchestration.
- Business orchestration belongs to Service; data access details belong to Repository/DAO.

### Database Conventions

- Use `config_` table prefix and `snake_case` naming.
- Use `BigInteger` primary keys via `AbstractAutoGeneratedIdAuditable<BigInteger>`.
- Prefer `is_xxx` boolean columns mapped explicitly with `@TableField`.
- No JSON column `typeHandler` is currently used; if JSON columns are introduced, explicit `typeHandler` is required.

### Tooling Conventions

- Use MyBatis-Plus `QueryWrapper` / `LambdaQueryWrapper` for query construction.
- Use MapStruct for DTO/Command to domain conversion.
- Use SLF4J for all logging.

### Engineering Red Lines

- Do not swallow exceptions (empty catch or ignore-only comments).
- Do not use `e.printStackTrace()`.
- Do not perform row-by-row DB access in loops (N+1).
- Do not log sensitive values in plaintext (signature, secret key, nonce).
- Use `CommonErrors` for consistent error semantics.

## Common API Examples

The examples below assume the service is running at `http://localhost:8080`.

### 1. Query config items with pagination

```bash
curl -G 'http://localhost:8080/api/v1/configItems' \
  --data-urlencode 'pageNo=1' \
  --data-urlencode 'pageSize=10'
```

### 2. Create a config item

```bash
curl -X POST 'http://localhost:8080/api/v1/configItems' \
  -H 'Content-Type: application/json' \
  -d '{
    "configEnvironmentId": 10001,
    "namespaceId": "application.yml",
    "formatType": "YML",
    "type": "PRIVATE",
    "content": "server:\n  port: 8081",
    "operationType": "ADD"
  }'
```

### 3. Update config content

```bash
curl -X PUT 'http://localhost:8080/api/v1/configItems/123/content' \
  -H 'Content-Type: application/json' \
  -d '{
    "content": "server:\n  port: 8082",
    "operationType": "UPDATE"
  }'
```

### 4. Release a config item

```bash
curl -X PUT 'http://localhost:8080/api/v1/configItemReleaseHistories/release' \
  -H 'Content-Type: application/json' \
  -d '{
    "releaseName": "release-2026-03-10",
    "configItemId": 123,
    "oldContent": "server:\n  port: 8081",
    "content": "server:\n  port: 8082",
    "releaseType": "NORMAL",
    "distributionStrategy": "IMMEDIATE_RELEASE"
  }'
```

### 5. Subscribe via long-polling

This endpoint requires signature headers: `X-Ca-Key`, `X-Ca-Timestamp`, `X-Ca-Nonce`, `X-Ca-Signature`.

```bash
curl -G 'http://localhost:8080/api/v1/configItems/subscribe' \
  -H 'X-Ca-Key: <accessKey>' \
  -H 'X-Ca-Timestamp: <timestamp_ms>' \
  -H 'X-Ca-Nonce: <nonce>' \
  -H 'X-Ca-Signature: <signature>' \
  --data-urlencode 'env=dev' \
  --data-urlencode 'componentCode=payment-service' \
  --data-urlencode 'namespace=application.yml'
```

### 6. Clone namespaces

```bash
curl -X POST 'http://localhost:8080/api/v1/configNamespaces/clone' \
  -H 'Content-Type: application/json' \
  -d '{
    "sourceEnvironmentId": 10001,
    "targetEnvironmentIds": [10002, 10003],
    "cloneMode": "OVERWRITE_FILES"
  }'
```

### 7. Sync namespaces

```bash
curl -X POST 'http://localhost:8080/api/v1/configNamespaces/sync' \
  -H 'Content-Type: application/json' \
  -d '{
    "sourceConfigItemId": 123,
    "targetEnvironmentId": 10002,
    "targetNamespaceIds": ["application.yml", "db.yml"],
    "conflictStrategy": "OVERWRITE_FILES"
  }'
```
