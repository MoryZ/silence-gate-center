# Silence Auth Center - Project Development Standards

## 1. Architecture Pattern

### 1.1 Layered Architecture

This project adopts the classic DDD (Domain-Driven Design) layered architecture:

```
api (Interface Layer)
├── Resource: REST API Controllers
├── assembler: DTO/VO Converters (using MapStruct)
└── config: API Configuration

domain (Domain Layer)
├── model: Domain Entities
├── service: Domain Services
└── repository: Repository Interfaces

infrastructure (Infrastructure Layer)
├── persistence: Persistence Implementation
│   ├── dao: MyBatis Mappers
│   └── *MyBatisRepository: Repository Implementation
└── message: Message Definitions

dto: Data Transfer Objects
├── *Command: Command Objects (Create/Update)
├── *Query: Query Objects
└── *DTO: Data Transfer Objects

vo: View Objects
security: Security Authentication
enums: Enum Definitions
```

### 1.2 Layer Responsibilities

#### Resource Layer (Interface Layer)

- **Responsibility**: Receive HTTP requests, validate parameters, call Service, return responses
- **Naming Convention**: `*Resource.java`
- **Core Annotations**: `@RestController`, `@RequestMapping("/api/v1")`
- **Permission Control**: Use `@PreAuthorize("hasAuthority('system:xxx:xxx')")`

**Example**:

```java
@RestController
@RequestMapping("/api/v1")
public class UserResource {
    private final UserMapper userMapper;
    private final UserService userService;

    @GetMapping("/users/{id}")
    @PreAuthorize("hasAuthority('system:user:list')")
    public User getUserById(@PathVariable BigInteger id) {
        return userService.findById(id);
    }
}
```

#### Service Layer (Domain Service)

- **Responsibility**: Business logic processing, transaction control, calling Repository
- **Naming Convention**: `*Service.java`
- **Core Annotations**: `@Service`, `@Transactional(rollbackFor = Exception.class)`
- **Transaction Specification**: All methods involving data modification must have transaction annotations

**Example**:

```java
@Service
public class UserService {
    private final UserDao userDao;

    @Transactional(rollbackFor = Exception.class)
    public BigInteger create(User user) {
        // Business logic
        userDao.insert(user);
        return user.getId();
    }
}
```

#### Repository Layer (Warehouse)

- **Interface Definition**: `domain/repository/*Repository.java`
- **Implementation Class**: `infrastructure/persistence/*MyBatisRepository.java`
- **Core Annotations**: `@Repository`
- **Responsibility**: Encapsulate data access logic, shield persistence technology details

**Example**:

```java
// Interface
public interface UserRepository {
    User findById(BigInteger id);
    int create(User user);
}

// Implementation
@Repository
public class UserMyBatisRepository implements UserRepository {
    private final UserDao userDao;

    @Override
    public User findById(BigInteger id) {
        return userDao.selectById(id);
    }
}
```

#### DAO Layer (Data Access)

- **Naming Convention**: `*Dao.java`
- **Core Annotations**: `@Mapper`
- **Tech Stack**: MyBatis-Plus BaseMapper
- **Custom SQL**: Use `@Select`, `@Update`, `@Delete` annotations

**Example**:

```java
@Mapper
public interface UserDao extends BaseMapper<User> {
    @Select("select * from sys_user where username = #{username} and status = #{status}")
    User findByUsernameAndStatus(String username, Boolean status);
}
```

## 2. Database Standards

### 2.1 Table Naming Rules

- **System Table Prefix**: `sys_` (e.g., `sys_user`, `sys_role`, `sys_menu`)
- **Business Table Prefix**: According to module definition (e.g., notification table uses `notice`)
- **Naming Style**: snake_case

### 2.2 Primary Key Design

- **Primary Key Type**: `BigInteger`
- **Generation Strategy**: Auto-generation (inheriting `AbstractAutoGeneratedIdAuditable`)
- **Field Name**: Unified use of `id`

### 2.3 Required Fields (Audit Fields)

#### Basic Audit Fields

All entity classes inherit `AbstractAutoGeneratedIdAuditable<BigInteger>` or `AbstractAutoGeneratedIdLogicalDeletionAuditable<BigInteger, Boolean>`, automatically including:

- `id`: Primary key
- `created_date` / `createdDate`: Creation time
- `created_by` / `createdBy`: Creator
- `last_modified_date` / `lastModifiedDate`: Last modification time
- `last_modified_by` / `lastModifiedBy`: Last modifier

#### Logical Deletion Fields

- `deleted` / `is_deleted`: Logical deletion flag (Boolean type, 0-not deleted, 1-deleted)

### 2.4 Field Type Mapping

| Java Type | Database Type | Example Fields |
|-----------|---------------|----------------|
| `BigInteger` | `BIGINT` | id, user_id |
| `String` | `VARCHAR` | username, name |
| `Boolean` | `TINYINT(1)` | status, deleted |
| `Map<String, Object>` | `JSON` | meta (menu metadata) |
| `Enum` | `VARCHAR` | type (using enum values) |

### 2.5 JSON Field Handling

Use MyBatis-Plus type handler:

```java
@TableField(typeHandler = StringToMapHandler.class)
private Map<String, Object> meta;
```

### 2.6 Field Existence Control

Non-database fields use `@TableField(exist = false)`:

```java
@TableField(exist = false)
private Set<BigInteger> roleIds;
```

## 3. Common Tech Stack

### 3.1 Core Dependencies

#### ORM Framework

- **MyBatis-Plus**: Database access enhancement tool
  - `BaseMapper`: Basic CRUD operations
  - `LambdaQueryWrapper`: Type-safe query builder
  - `Page`: Pagination object

**Usage Example**:

```java
// Pagination query
Page<User> page = new Page<>(pageNo, pageSize);
userDao.selectPage(page, new LambdaQueryWrapper<User>()
    .eq(User::getStatus, true)
    .orderByDesc(User::getCreatedDate));
```

#### Object Conversion

- **MapStruct**: Object mapping framework
  - Interface inherits `Converter<Source, Target>`
  - Configuration class: `AuthCenterMapStructSpringConfig`

**Usage Example**:

```java
@Mapper(uses = AuthCenterMapStructSpringConfig.class)
public interface UserMapper extends Converter<UserCommand, User> {
    User convert(UserCommand userCommand);
    UserVo toUserVo(User user);
}
```

#### Security Authentication

- **Spring Security**: Permission control
- **JWT (Auth0)**: Token generation and verification
  - Secret key: `SecurityConstants.TOKEN_SECRET`
  - Prefix: `Bearer `

#### Query Builder

- **Custom QueryWrapper Converter**: `QueryWrapperConverter`
  - Supports `@RelationalQueryProperty` annotation
  - Automatically converts Query objects to MyBatis-Plus query conditions

**Usage Example**:

```java
public class UserQuery {
    @RelationalQueryProperty(type = Part.Type.STARTING_WITH)
    private String username;

    @RelationalQueryProperty(type = Part.Type.SIMPLE_PROPERTY)
    private Boolean status;
}

// Usage in Controller
var queryWrapper = QueryWrapperConverter.convert(query, User.class);
userService.query(page, queryWrapper);
```

### 3.2 Utility Classes

#### Collection Utilities

- **CollectionUtils**: Collection conversion utilities
  - `transformToList`: List conversion

#### Tree Structure

- **TreeFormatUtils**: Tree data processing
  - `listToTree`: List to tree structure

### 3.3.2 Error Message Definition

All system error messages are defined in the `AuthCenterMessages` enum and internationalized through properties files.

**Example**:

```java
public enum AuthCenterMessages implements ErrorCodedEnumMessageSourceResolvable {
    USER_NOT_EXIST(HttpStatus.BAD_REQUEST, 51),
    PASSWORD_NOT_CORRECT(HttpStatus.BAD_REQUEST, 52),
    USERNAME_ALREADY_EXIST(HttpStatus.BAD_REQUEST, 59),
    // ...
}
```

#### Corresponding Internationalization Files

**English** (silence_auth_center_messages.properties):

```properties
AuthCenterMessages.UserNotExist=User does not exist
AuthCenterMessages.PasswordNotCorrect=Password is incorrect
AuthCenterMessages.UsernameAlreadyExist=Username already exists
```

**Chinese** (silence_auth_center_messages_zh_CN.properties):

```properties
AuthCenterMessages.UserNotExist=用户不存在
AuthCenterMessages.PasswordNotCorrect=密码错误
AuthCenterMessages.UsernameAlreadyExist=用户名已存在
```

### 3.3.3 Usage Standards

#### Must Follow

1. **All business exceptions must use internationalized messages**

```java
// Correct approach
if (user == null || user.getDeleted()) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}

// Incorrect approach
if (user == null || user.getDeleted()) {
    throw new BusinessException("User does not exist"); // Hardcoded string
}
```

2. **When adding new exception constants, must update properties files simultaneously**

```java
// 1. Add in AuthCenterMessages.java
USERNAME_ALREADY_EXIST(HttpStatus.BAD_REQUEST, 59),

// 2. Add in silence_auth_center_messages.properties
AuthCenterMessages.UsernameAlreadyExist=Username already exists

// 3. Add in silence_auth_center_messages_zh_CN.properties
AuthCenterMessages.UsernameAlreadyExist=用户名已存在
```

3. **Exception messages with specific parameters use method parameters**

```java
// Correct approach - parameters passed at runtime
throw AuthCenterMessages.USER_NOT_EXIST.createException(username);

// Supported parameter methods
createException()                    // No parameters
createException(Object arg)          // Single parameter
createException(Object... args)      // Multiple parameters
```

4. **Error code range planning**

| Module | Code Range | Example |
|--------|------------|---------|
| User Management | 51-60 | USER_NOT_EXIST (51) |
| Role Management | 61-70 | ROLE_NOT_EXIST (57) |
| Menu Management | 71-80 | MENU_NOT_EXIST (54) |
| Authentication | 81-90 | TOKEN_INVALID (81) |
| Other Business | 91+ | As needed |

#### Prohibited Items

1. **Do not hardcode Chinese or English strings**

```java
// Incorrect
throw new BusinessException("User does not exist");
throw new BusinessException("User not exist");

// Correct
throw AuthCenterMessages.USER_NOT_EXIST.createException();
```

2. **Do not mix internationalization and hardcoding**

```java
// Incorrect mixing
if (error) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}
if (other) {
    throw new BusinessException("Other error"); // This is hardcoded
}

// Correct - all use internationalization
if (error) {
    throw AuthCenterMessages.USER_NOT_EXIST.createException();
}
if (other) {
    throw AuthCenterMessages.OTHER_ERROR.createException();
}
```

3. **Do not concatenate error messages in code**

```java
// Incorrect
String msg = "User " + username + " not found";
throw new BusinessException(msg);

// Correct
throw AuthCenterMessages.USER_NOT_EXIST.createException(username);
```

### 3.3.4 i18n Message File Maintenance Checklist

- [ ] When adding new exception constants, update all three properties files simultaneously
- [ ] Keep English and Chinese translations logically consistent
- [ ] No duplicate error code ranges
- [ ] Keep properties files in UTF-8 encoding (with Chinese)
- [ ] Regularly review unused message constants

## 4. Development Red Lines

### 4.1 Database Operations

#### Prohibited Items

1. **Prohibited from querying database in loops**

```java
// Incorrect example
for (BigInteger userId : userIds) {
    User user = userDao.selectById(userId); // Query database in each loop
}

// Correct example
List<User> users = userDao.selectBatchIds(userIds); // Batch query
```

2. **Prohibited from querying large amounts of data without pagination**

```java
// Incorrect example
List<User> users = userDao.selectList(null); // Query all data

// Correct example
Page<User> page = new Page<>(pageNo, pageSize);
userDao.selectPage(page, queryWrapper);
```

3. **Prohibited from using `select *` (MyBatis-Plus queries all fields by default, be careful)**

```java
// To specify fields, use LambdaQueryWrapper
new LambdaQueryWrapper<User>()
    .select(User::getId, User::getUsername);
```

#### Must Follow

1. **All modification operations must have transaction annotations**

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    // Update logic
}
```

2. **Related data modifications must be in the same transaction**

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    userDao.updateById(user);
    userRoleDao.delete(new LambdaQueryWrapper<UserRole>()
        .eq(UserRole::getUserId, user.getId()));
    // Reassign roles
    assignUserRoles(user.getId(), user.getRoleIds());
}
```

### 4.2 Exception Handling

#### Prohibited Items

1. **Prohibited from swallowing exceptions**

```java
// Incorrect example
try {
    // Business logic
} catch (Exception e) {
    // Do nothing
}

// Correct example
try {
    // Business logic
} catch (Exception e) {
    log.error("Business processing failed", e);
    throw new RuntimeException("Business processing failed", e);
}
```

2. **Prohibited from only printing logs without throwing after catching exceptions**

```java
// Incorrect example
try {
    userDao.insert(user);
} catch (Exception e) {
    log.error("User creation failed", e);
    // Exception not thrown, caller cannot perceive
}

// Correct example
try {
    userDao.insert(user);
} catch (Exception e) {
    log.error("User creation failed", e);
    throw AuthCenterMessages.USER_CREATE_FAILED.createException("User creation failed", e);
}
```

#### Must Follow

1. **Use unified business exceptions**

```java
if (existingRole == null || existingRole.getDeleted()) {
    throw AuthCenterMessages.ROLE_NOT_EXIST.createException("Role does not exist");
}
```

2. **Transaction methods must declare `rollbackFor = Exception.class`**

```java
@Transactional(rollbackFor = Exception.class)
public void delete(BigInteger id) {
    // Business logic
}
```

### 4.3 Code Standards

#### Prohibited Items

1. **Prohibited from hardcoding strings**

```java
// Incorrect example
if ("admin".equals(username)) { }

// Correct example
if (SecurityConstants.ADMIN_USERNAME.equals(username)) { }
```

2. **Prohibited from using magic numbers**

```java
// Incorrect example
user.setStatus(1);

// Correct example
user.setStatus(true); // Or use enum
```

3. **Prohibited from directly injecting DAO in Service layer**

```java
// Incorrect example (Service layer directly injects DAO)
@Service
public class UserService {
    private final UserDao userDao; // Should inject Repository
}

// Correct example
@Service
public class UserService {
    private final UserRepository userRepository;
}
```

#### Must Follow

1. **Entity classes must inherit audit base class**

```java
public class User extends AbstractAutoGeneratedIdLogicalDeletionAuditable<BigInteger, Boolean> {
    // Field definitions
}
```

2. **DTO conversion must use MapStruct**

```java
@Mapper(uses = AuthCenterMapStructSpringConfig.class)
public interface UserMapper extends Converter<UserCommand, User> {
    User convert(UserCommand userCommand);
}
```

3. **Query objects must use `@RelationalQueryProperty` annotation**

```java
public class UserQuery {
    @RelationalQueryProperty(type = Part.Type.STARTING_WITH)
    private String username;
}
```

4. **Logical deletion must use flags, prohibit physical deletion**

```java
@Transactional(rollbackFor = Exception.class)
public void delete(BigInteger id) {
    User user = userDao.selectById(id);
    user.setDeleted(true);
    userDao.updateById(user);
}
```

### 4.4 Security Standards

#### Prohibited Items

1. **Prohibited from storing passwords in plaintext**

```java
// Must be encrypted storage, be careful with default passwords in examples for production
user.setPassword(passwordEncoder.encode("123456"));
```

2. **Prohibited from printing sensitive information in logs**

```java
// Incorrect example
log.info("User login: {}", user); // user object contains password

// Correct example
log.info("User login: username={}", user.getUsername());
```

3. **Prohibited from directly returning entity classes**

```java
// Incorrect example
@GetMapping("/users/{id}")
public User getUser(@PathVariable BigInteger id) {
    return userService.findById(id);
}

// Correct example
@GetMapping("/users/{id}")
public UserVo getUser(@PathVariable BigInteger id) {
    User user = userService.findById(id);
    return userMapper.toUserVo(user);
}
```

#### Must Follow

1. **All interfaces must have permission control**

```java
@PreAuthorize("hasAuthority('system:user:list')")
@GetMapping("/users/{id}")
public User getUserById(@PathVariable BigInteger id) {
    return userService.findById(id);
}
```

2. **Token verification must use unified authentication mechanism**

```java
// Use SilenceHallServerTokenAuthority for Token issuance and verification
var token = silenceHallServerTokenAuthority.issueToken(principal);
```

## 5. Best Practices

### 5.1 Pagination Query Template

```java
@GetMapping(value = "/users", params = {"pageNo", "pageSize"})
public Page<User> query(Page<User> page, UserQuery query) {
    var queryWrapper = QueryWrapperConverter.convert(query, User.class);
    return userService.query(page, queryWrapper);
}
```

### 5.2 Batch Operation Template

```java
@Transactional(rollbackFor = Exception.class)
private void assignUserRoles(BigInteger userId, List<BigInteger> roleIds) {
    List<UserRole> userRoles = roleIds.stream()
        .map(roleId -> {
            UserRole userRole = new UserRole();
            userRole.setUserId(userId);
            userRole.setRoleId(roleId);
            return userRole;
        }).collect(Collectors.toList());
    userRoleDao.insertBatchSomeColumn(userRoles);
}
```

### 5.3 Tree Structure Build Template

```java
private List<MenuDto> buildMenuTree(List<Menu> menus) {
    var menuMapper = Mappers.getMapper(MenuMapper.class);
    var menuDtos = menus.stream()
        .map(menuMapper::convertToDto)
        .collect(Collectors.toList());

    Map<BigInteger, List<MenuDto>> parentMap = menuDtos.stream()
        .collect(Collectors.groupingBy(MenuDto::getParentId));

    menuDtos.forEach(menu ->
        menu.setChildren(parentMap.getOrDefault(menu.getId(), new ArrayList<>())));

    return parentMap.getOrDefault(BigInteger.ZERO, new ArrayList<>());
}
```

### 5.4 Parameter Validation Template

```java
@Transactional(rollbackFor = Exception.class)
public void update(User user) {
    // 1. Check if entity exists
    User existingUser = userDao.selectById(user.getId());
    if (existingUser == null || existingUser.getDeleted()) {
        throw AuthCenterMessages.USER_NOT_EXIST.createException("User does not exist");
    }

    // 2. Business rule validation
    if (!existingUser.getUsername().equals(user.getUsername())
        && isUsernameExists(user.getUsername())) {
        throw AuthCenterMessages.USERNAME_ALREADY_EXIST.createException("Username already exists");
    }

    // 3. Execute update
    userDao.updateById(user);
}
```

## 7. Notes

1. **All database entity classes must inherit audit base class**, automatically filling creation time, modification time and other fields
2. **Logical deletion takes priority over physical deletion**, ensuring data traceability
3. **Transaction boundaries are clear**, Service layer methods are transaction boundaries
4. **Interface return values use VO objects**, avoiding direct exposure of entity classes
5. **Query conditions use Query objects**, with `@RelationalQueryProperty` annotation for automatic query building
6. **Permission control is granular to button level**, using `system:module:operation` format
7. **Sensitive information such as passwords must be encrypted**
8. **All external inputs must be validated**
